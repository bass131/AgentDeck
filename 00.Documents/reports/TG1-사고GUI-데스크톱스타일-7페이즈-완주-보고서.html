요약 — 무엇을 만들었나 · 왜 필요한가

무엇을 만들었나 — Claude Code CLI로 하던 사고(thinking) 진행 표시를 AgentDeck GUI 안에서 Claude Code Desktop 스타일로 재작업했습니다. 끝난 뒤 두 가지가 성립합니다. ① 턴 블록 통합 — 한 턴 = 한 블록 = 아바타 1개. 예전엔 사고 상태(스피너)·사고 전문·답변 버블이 각자 아바타를 단 독립 .msg 블록 3개로 나열됐고 아바타 아이콘조차 불일치했습니다(사고=IconClaude·답변=IconSpark). 이 셋을 groupIntoTurnBlocks로 같은 블록의 좌측 연속 거터로 꿰어, 사고에서 답변으로 넘어갈 때 블록이 바뀌지 않고 같은 블록 안에서 이어집니다. ② 한 줄 상태 라인 — 그 블록 안에 StatusLine("✻ 궁리하는 중… (12s · ↑ 3.4k tokens)")이 사고 신호(심볼 애니메이션·유희적 동사 순환·경과 초·실시간 토큰)를 한 줄로 통합하고, 답변이 시작되면 소멸합니다. 여기에 ③ 공식 Claude Spark 아바타를 화자 아바타로 얹어(자체 SVG 대신), 듀얼 백엔드에서 "어느 엔진이 말하는가"가 아바타로 구분됩니다.

왜 필요한가 — GAP1 P16이 사고 인디케이터와 답변의 "분리감"을 인접 연출(gap 축소)로 임시 봉합했지만, 그건 과도기 처리였습니다. 분리감의 근본 원인은 요소의 결함이 아니라 "턴"이라는 묶음 개념이 DOM에 없다는 것이었고(P16 학습 계승), 그래서 진짜 해법은 구조 — 턴 블록이라는 DOM 묶음을 만드는 것입니다. 영호가 GAP1 마감 육안 중 낸 피드백 2건에서 이월된 이 마일스톤이 그 구조를 세웠습니다. 상태 라인 통합은 사고 중 화면 소음(스피너·전문·토큰 게이지가 따로 노는 것)을 한 줄로 정돈해 Claude Code Desktop 수준의 밀도를 회복하는 것이 목적입니다.
