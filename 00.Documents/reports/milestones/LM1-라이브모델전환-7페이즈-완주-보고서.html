모드 전환(setPermissionMode)은 setter 후 상태 이벤트로 반영을 확인할 수 있지만, 모델 전환은 SDK가 역통지 이벤트를 주지 않습니다. 그래서 반영 정본 = 다음 assistant message.model(관측 정본) · reject 조건부 롤백 · 재사용 경로 change-guard 안전망으로 대체했습니다. model-fallback 불가침(_currentModel = 사용자 의도값, SDK 폴백값으로 덮지 않음)은 이 비대칭 하에서 사용자 의도를 지키는 핵심입니다. effort는 라이브 모델 전환과 달리 SDK 라이브 API가 없어 세션 생성 시 1회 고정되며(P05 프로브 실측), 그래서 피커 고지가 "새 대화(세션)부터 적용"입니다. 한편 shared/model-effort.ts는 IPC 채널이 아니라 순수 도메인 상수 모듈이라 계약 버전 bump가 아닙니다 — 역의존 0이며, 키 3자 정합은 테스트 파일에서만 main을 import해 단언합니다(번들 무관).
