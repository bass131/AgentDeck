02

Laptop 최초 셋업 체크리스트

순서대로 한 번만. 각 스텝 끝의 GATE로 통과 여부를 확인하고 다음으로.

  1. 설치물 3종 준비

    Git for Windows(표준 설치 경로) · Node.js v24.x · Claude Code CLI. Desktop 실측 기준은 Node v24.15.0 / npm 11.13.0 — 같은 major(24)면 됩니다. 

    # Node 설치 후 확인
node --version   # v24.x 인지
git --version

# Claude Code CLI 설치 + 로그인
npm install -g @anthropic-ai/claude-code
claude login

    주의: Git은 반드시 표준 설치로 — AgentDeck의 훅(hook)들이 Git Bash에서 실행되는데, 표준 설치는 Claude Code가 자동 탐지합니다. portable 설치를 썼다면 사용자 환경변수에 CLAUDE_CODE_GIT_BASH_PATH를 지정해야 합니다.

  2. 클론 — 브랜치를 지정해서

    cd C:\Dev
git clone -b feature/gap1-core-parity https://github.com/bass131/AgentDeck.git
cd AgentDeck

    -b 옵션은 클론 직후 체크아웃할 브랜치를 지정합니다. 깜빡했다면 클론 뒤에 git checkout feature/gap1-core-parity를 실행하면 동일합니다 (원격 추적 브랜치가 자동으로 연결됩니다).

    GATE git branch --show-current 출력이 feature/gap1-core-parity.

  3. .env 수동 이동 — git 밖 유일한 짐

    시크릿은 형상관리에 넣지 않는 것이 규칙(CORE-03)이라 .env는 클론에 딸려오지 않습니다. Desktop의 프로젝트 루트에서 Laptop의 프로젝트 루트로 USB 등 오프라인 수단으로 직접 복사하세요.

    금지: 메신저·메일·클라우드 공유로 보내는 것. 시크릿이 서버에 남습니다.

  4. 의존성 설치 + 회귀 게이트로 검증

    npm install
npm run typecheck
npm run test

    native 모듈(sqlite 등)을 전부 제거한 상태라 빌드 도구 없이 npm install만으로 끝납니다. typecheck·test가 초록이면 "코드가 완전한 상태로 건너왔다"는 기계 증명입니다.

    GATE typecheck 에러 0 · 테스트 fail 0. 하나라도 빨강이면 진행하지 말고 Desktop 쪽 마지막 커밋을 확인.

  5. Claude Code 첫 실행 — 신뢰 2회 + 플러그인

    프로젝트 폴더에서 claude 실행 → ① 폴더 신뢰 확인 → ② 훅 신뢰 (/hooks에서 확인 — 설정 다이제스트 기준이라 최초 1회면 됩니다). 그다음 플러그인 2종을 설치합니다:

    # Claude Code 안에서
/plugin   # → openai-codex, frontend-design(claude-plugins-official) 설치

    플러그인·MCP 연결은 머신 로컬입니다(~/.claude 아래 저장). git으로 오지 않으므로 Laptop에서 한 번 다시 설치합니다. Codex CLI를 쓸 거면 별도 설치 + codex login도 필요합니다.

  6. 세션 시작 — 좌표는 pin이 알고 있다

    # Claude Code 안에서
/session:start

    work-pin(.claude/state/current-pin.txt)이 커밋에 포함돼 건너오기 때문에, Laptop의 Claude도 "지금 어느 Phase, 다음 액션이 뭔지"를 즉시 압니다. 이것이 멀티머신 체제의 핵심 장치입니다.

    GATE 세션 시작 응답에 현재 Phase 좌표가 표시되면 셋업 완료. 이후는 평소처럼.
03

왕복 루틴 카드

최초 셋업 이후 매번 반복하는 것은 이 두 장뿐입니다. 방향은 Desktop→Laptop, Laptop→Desktop 어느 쪽이든 동일합니다.

DEPARTURE — 떠날 때 COMMIT & PUSH
  1. 작업을 마디 단위로 커밋 — 게이트(typecheck·test) 통과 후, 명시 파일만 staging.
  2. 마디가 안 나오면 wip: 커밋 — 메시지에 "게이트 미실행"을 명시.
  3. pin(.claude/state/current-pin.txt) 갱신을 커밋에 포함.
  4. git push — 푸시된 것만 반대편에 존재한다.

push를 잊으면 반대편 머신에서는 그 작업이 존재하지 않습니다. "떠날 때 push"가 이 체제의 전부입니다.

ARRIVAL — 앉을 때 PULL & START
  1. git pull --ff-only — 빨리 감기만 허용. 실패하면 §06 참고, 억지로 풀지 말 것.
  2. claude 실행 → /session:start.
  3. pin 좌표(현재 Phase·다음 액션) 확인 후 이어서 작업.

--ff-only는 "로컬에 다른 커밋이 없을 때만 받아라"는 뜻 — 두 머신이 갈라진 걸 조용히 섞지 않고 즉시 드러내는 안전장치입니다.

04

NO-GO — 절대 하지 않는 것

멀티머신에서 아래 행위는 단일 머신 때보다 피해가 큽니다. 반대편 머신의 이력과 충돌하기 때문입니다.

  • NO-GO이력 재작성·force-push (rebase 후 push, push --force) — 반대편 머신이 이미 받아 간 이력을 부수면 복구가 매우 어렵습니다.
  • NO-GOstash를 이동 수단으로 쓰기 — stash는 그 머신의 로컬 임시 보관함일 뿐, push되지 않습니다. 이동은 오직 커밋+push.
  • NO-GOmaster에서 직접 작업 — master는 PR 머지 대상. 작업은 항상 feature 브랜치에서.
  • NO-GOgit add . 일괄 스테이징 — 의도하지 않은 파일(로컬 실험물)이 섞여 반대편으로 넘어갑니다. 명시 파일만.
  • NO-GO.env를 커밋·메신저·클라우드로 옮기기 — 시크릿은 오프라인 수동 이동만 (CORE-03).
05

git 밖에서 사는 것들

커밋으로 건너오지 않는 것의 전체 목록 — "왜 Laptop에 이게 없지?"의 답은 전부 여기 있습니다.

항목위치Laptop에서의 처리
.env (시크릿) 프로젝트 루트 (gitignore) USB 등으로 수동 복사 — §02 스텝 03
Claude 메모리·세션 기록 ~/.claude/projects/C--Dev-AgentDeck/ 머신마다 따로 쌓임(기본). 옮기고 싶으면 memory/ 폴더만 수동 복사 가능 — 필수 아님. 작업 좌표는 pin이 대신 전달

node_modules / 빌드 산출물 프로젝트 내 (gitignore) npm install이 재생성 — 옮기지 말 것
반대로, 이미 git에 승격돼 자동으로 건너오는 것: work-pin(.claude/state/current-pin.txt) · tdd-enforce 상태 · 00.Documents/Artifacts 스크린샷. 2026-07-14에 멀티머신을 위해 형상관리로 승격했습니다 — 별도 조치 불필요.
06

트러블슈팅

git pull --ff-only가 거부된다 ("Not possible to fast-forward")

두 머신이 같은 브랜치에서 서로 다른 커밋을 만들었다는 뜻입니다(예: Desktop에서 push를 잊고 Laptop에서 새 커밋). 여기서 멈추는 게 정상 동작입니다 — 조용히 섞이는 것보다 낫습니다. 해소는 git merge origin/feature/gap1-core-parity(머지 커밋 허용)로 하되, rebase는 이력 재작성이라 금지. 확신이 없으면 두 머신의 git log --oneline -5를 놓고 Claude와 함께 정리하세요.

훅이 실행되지 않거나 훅 에러가 난다

대부분 Git Bash 경로 문제입니다. Git for Windows가 표준 경로에 설치됐는지 확인하고, portable 설치라면 CLAUDE_CODE_GIT_BASH_PATH 환경변수를 지정하세요. 줄바꿈은 .gitattributes가 .claude/hooks/**를 LF로 고정하므로 클론만 정상이면 문제없습니다.

Laptop의 Claude가 이전 대화·결정을 모른다

정상입니다 — 대화 기록과 자동 메모리는 머신 로컬입니다. 대신 작업 좌표는 pin이, 결정 이력은 ADR과 .claude/CHANGELOG.md가, Phase 상태는 01.Phases/가 커밋으로 건너옵니다. /session:start가 이것들을 읽고 맥락을 복원합니다.

npm install에서 네이티브 빌드 에러가 난다

현재 의존성에는 네이티브 모듈이 없어(sqlite 제거, ADR-006 superseded) 원칙적으로 발생하지 않습니다. 난다면 Node 버전이 v24.x인지부터 확인하세요 (node --version).
