## 🎯 목표

**사고 중 GUI를 Claude Code Desktop 스타일로 재작업**한다. 끝나면 두 가지가 성립한다.

1. **턴 블록 통합** — 한 턴 = 한 블록 = 아바타 1개. 지금은 사고 상태(WorkingIndicator 스피너)·사고 전문(ThinkingItem)·답변 버블이 각자 아바타를 단 독립 `.msg` 블록 3개로 나열되고 아바타 아이콘조차 불일치한다(사고=IconClaude·답변=IconSpark). 이 셋을 같은 블록의 좌측 연속 거터(│)로 꿰어, 사고에서 답변으로 넘어갈 때 블록이 바뀌지 않고 같은 블록 안에서 이어진다. 그 안에 **한 줄 상태 라인**("✻ 궁리하는 중… (12s · ↑ 3.4k tokens)")이 사고 신호(심볼 애니메이션·유희적 동사 순환·경과 초·실시간 토큰)를 통합하고, 답변이 시작되면 소멸한다.
2. **Claude 공식 pinwheel 로고 아바타** — 자체 SVG(IconClaude/IconSpark) 대신 Claude 공식 로고를 화자 아바타로 쓴다. 듀얼 백엔드(Claude/Codex)에서 화자 아바타 = "어느 엔진이 말하는가"를 구분하는 기능 정보다. Claude 엔진 한정 교체(Codex 아바타는 기존 유지).

- ① **대화 내 엔진 아바타로만 사용** — 앱 자체 아이콘·이름에는 Claude 로고 **사용 금지**(파트너십·후원 오인 위험이 대화 내 배지와 성격이 다름).

## 🔗 의존성 그래프

```
P01 ──▶ P02 ─────┐
   │             ├──▶ P04 ──┐
   ├──▶ P03 ─────┘          ├──▶ P06 ──▶ P07
   │      └─────────────────┤
   └──▶ P05 ────────────────┘
```

- **P01 = 유일한 착수점** — 구현 착수 전 3대 불확실성 제거(좌표 최신화·공식 에셋 확보·셀렉터 영향 census). 코드 무변경(조사 + 에셋 파일 1개 추가만). P02·P03·P05가 전부 P01의 산출물(좌표표·census·에셋)에 의존한다.
- **P04 = P02(경과 초 데이터) + P03(턴 블록)** — 한 줄 상태 라인은 경과 초 데이터 토대(P02)와 턴 블록 구조(P03) 둘 다 위에 얹힌다.
- **P06 = P03·P04·P05** — 표면 전파는 단일챗에서 완성된 턴 블록·상태 라인(P03·P04)을 멀티패널·서브에이전트에 옮기고, 서브 표면은 P05 계약 확장 결과에 종속(토큰·훅 배지).
- **P07 = P03~P06 (최후 마감 게이트)** — 셀렉터 정합 최종 스윕·시각검증 채증·라이브 배터리·영호 육안 패키지.

### 병렬 가능성 요약

- **P02 ∥ P05** (P01 후) — 둘은 서로 독립. P02는 renderer store, P05는 cross(계약).
- **P03 ∥ P05** — 턴 블록 재구조(renderer)와 SubAgent 계약 확장(cross)은 서로 독립 레인. P05는 P02~P04 전 구간과 병렬 가능(브리프 실측: 서브 표면 계약 종속성은 P06에서 합류).
- 착수 시점: P01 단독. P01 완료 후 {P02, P03, P05} 병렬 착수 가능(P03은 human-visual 육안 병행 병목).

## 🗂️ P16 자산 처분표 (GAP1 P16 = 과도기 처리)

턴 블록 통합은 P16 인접 연출을 대체하되, P16이 만든 데이터·순수 함수 자산은 그대로 재사용한다.

- **재사용 (살릴 자산)**:
  - `deriveHookTurnBadges` 순수 함수 — 훅 배지 턴 귀속(차단 아이템 → 턴 경계 배타 구간 내 최근접 후속 assistant, 부재 시 선행, 전무 시 무귀속).
  - 훅 빨간 배지(차단·auto-deny 시각 표식).
  - `hookRuns` `runId` 배선.
  - `estimatedTokens` 데이터 — 한 줄 상태 라인의 토큰 카운트 소스.
- **대체 (턴 블록 통합이 밀어낼 자산)**:
  - 인접 연출 CSS — 블록 간 gap 축소·연결 언어. 세 블록을 시각적으로 붙여 보이게 하던 과도기 처리로, 턴 블록 통합(구조적 한 블록화)으로 대체된다(`.thread` gap 재구조 포함).

## ⚠️ 마일스톤 공통 주의

- **셀렉터 계약 파괴 = 최대 리스크** — `.msg` 등 셀렉터 계약을 변경하면 96개 시각·라이브 테스트에 영향이 간다(브리프 5단원 명제). 턴 블록 통합이 `.msg` 블록 구조 자체를 바꾸므로, **P01 census가 선행 게이트**다. census 밖 셀렉터 변경 발견 시 보고 후 중단.
- **P05 additive 원칙 = ADR-035 규율** — SubAgent 계약 확장은 옵셔널 필드만. 기존 필드 의미 변경 금지. SDK가 서브 사고 토큰을 실제로 주는지 **probe-first 채증 후 설계**(못 주면 "데이터 원천 부재"로 명시 보류 박제 + renderer 몫은 우아한 부재 처리 — 조용한 드롭 금지).
- **ui-visual Phase(P03·P04·P06·P07)** — 기능은 자율 진행하되 무인 commit X, 영호 육안 병행. UI 롤아웃 시 노출 지점 전수 열거(배지 3번째 지점 누락 교훈, memory: ui-rollout-surface-enumeration).
- **좌표 재실측 필수** — 브리프 4단원 renderer 좌표는 2026-07-15 스카우트 기준이라 이후 변동 가능. 재실측 없이 그대로 믿으면 안 된다(memory: verify-fixes-empirically, LR1 교훈).
- **TDD 공통** — 모든 Phase 실패 테스트 선행(CORE-05, tdd-guard 강제).
- **로고는 대화 아바타 한정** — 앱 아이콘·이름 금지(상표 게이트 ①).

## 📚 마일스톤 학습 테마

- 분리감은 요소의 결함이 아니라 "턴"이라는 묶음 개념의 DOM 부재다(P16 학습 계승) — 구조로 해소(P03).
- 렌더 경로 시간 의존(Date.now())의 주입 가능화 = 테스트 가능성(P02).
- additive 계약 확장 원칙 — 기존 소비처 무접촉으로 필드만 추가(P05, ADR-035).
- 노출 지점 전수 열거 = 표면 누락 방지(P06).
- 셀렉터 계약이 테스트 스위트의 암묵적 API다 — 구조 변경 = 계약 변경(P01·P07).
