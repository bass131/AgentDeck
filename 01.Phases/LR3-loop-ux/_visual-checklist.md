# LR3 육안 검증 투어 (영호용 — 약 10~15분, 라이브 토큰 소모 있음)

> 준비: `npm run dev` 로 앱 실행. 각 단계의 **입력**을 그대로 입력창에 치고, **기대**와
> 실제 화면을 비교. 어긋나면 단계 번호와 함께 스크린샷을 남겨주세요.
> ⚠ 루프를 만드는 단계(3·4)는 확인 후 **반드시 정지 버튼** — 안 끄면 백그라운드 토큰 소모.

## 0. 시작 상태 (P03 — 기본값)
- [ ] 새 대화 클릭. 컴포저의 **REPL 버튼이 기본 ON**인지 (P03: replMode 기본 true).
- [ ] 앱 재시작 후에도 토글 상태가 유지되는지 (prefs 영속 — OFF로 바꿨다 재시작해보면 OFF 유지).

## 1. 금색 REPL 표시등 (P06 — 영호 조정 2026-07-03 반영: ON=상시 점등)
- 기대: REPL 토글이 **ON이면 항상** 금색 형광 점등(은은한 숨쉬기 pulse 글로우, 2.6s 주기),
  OFF로 끄면 소등. 활동 여부와 무관 — "기능이 켜져 있다"는 표시등.
- 판단 포인트: 형광 세기·pulse 속도 과유불급 / 테라코타와의 조화 / **라이트·다크 둘 다**
  (라이트는 씻겨 보이지 않게 더 진하게 튜닝됨).

## 2. AUTO 세션 수명 — 맥락 연속 (P02)
- 입력 ①: `코드워드 MANGO7을 기억해. OK라고만 답해.`
- (응답 종료 후 — 이 사이 백그라운드에서 세션이 스스로 닫혔음: 실측 688ms)
- 입력 ②: `코드워드가 뭐였지?`
- 기대: **MANGO7 회상** — 턴 사이 프로세스가 닫혀도 대화 맥락은 resume으로 온전.
- (선택·심화) 작업관리자에서 턴 종료 몇 초 뒤 엔진 프로세스가 사라지는 것 확인 가능.

## 3. /loop → SDK 크론 + 배너 + 전체박스 gloss (P03+P04+P06)
- 입력: `/loop 1m 지금 시각을 아주 짧게 알려줘`
- 기대: ① 도구 카드(CronCreate)가 스레드에 뜨고 ② 하단에 **"loop 진행중" 배너**(회전 아이콘)
  ③ **채팅창+입력창 전체 박스**에 은은한 링(gloss — 예전처럼 스크롤 영역만 감싸지 않음)
  ④ REPL 버튼 금색 유지 ⑤ ~1분 뒤 틱이 새 응답으로 도착.
- ⚠ 크론 생성은 모델 재량이라 가끔 그냥 1회 답하고 끝남(실측 1/3 빈도) — 그러면 같은 문장 재전송.
- [ ] **정지 버튼(배너의 정지) 클릭 → "루프 정지됨 · 반복 실행이 멈췄어요 — 더 이상 자동
      호출되지 않아요" 확인 배너**(✕로 닫기) ← 신규(정지 신뢰 피드백).
- ℹ **CronList 기록 잔존은 정상**: 정지 후 "크론 제거 됐나?"라고 물으면 CronList가 옛
      크론을 목록에 보여줄 수 있다 — 크론 *기록*이 대화 이력에 남기 때문. 실행은 확실히
      멈춘다(2중 probe 실측: 정지 후 80s 증가 0 + resume 후 90s 자율 틱 0 —
      lr3-p06-stop-cleanup.e2e.ts 2종).

## 4. 자연어 루프 → self-paced 배너 (P04 — LR2까진 완전 비가시였던 경로)
- 입력: `이 작업을 주기적으로 반복해줘: 'PING'이라고만 답하기. 내가 멈추라고 할 때까지.`
- 기대: 권한 팝업(Skill 등) 1~2회 → 승인 → **"loop 진행중" 배너**(self-paced —
  크론이 아니라 ScheduleWakeup인데도 배너가 뜨는 게 P04의 핵심).
- [ ] 정지 버튼 → 소멸. ← 필수!

## 5. goal 진행 배너 (P06)
- 입력: `/goal 서로 다른 인사말 3개를 만들면 목표 달성`
- 기대: 하단에 **goal 배너**(회전 아이콘 + "N턴" 뱃지 — 턴이 늘며 숫자 증가), 스레드에는
  진행 카드가 남아 완료 시 "목표 반복을 마쳤어요 · N턴"으로 확정.
- 참고: loop와 goal이 동시일 땐 loop(sdk)가 우선 표시(단일 표시 — 동시 배너 2개 금지).

## 6. 멀티패널 전환-연속성 (P07)
- 멀티 에이전트 모드 → 패널 1에 입력: `1부터 30까지, 숫자마다 한 줄 코멘트를 달며 세줘`
- **스트리밍 중에** 단일 모드로 전환 → 5초쯤 기다렸다 멀티로 복귀.
- 기대: 패널 내용이 **비어있지 않고 이어져 있음**(전환 중 도착분 포함 — 예전엔 영구 증발).
- (역방향도) 단일챗에 같은 요청 → 실행 중 멀티로 전환 → 복귀 → 응답이 정상 완료돼 있고
  입력창이 다시 활성화되는지(예전엔 "실행 중"에 영구 고착).
