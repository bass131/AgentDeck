# UI_GUIDE — AgentDeck

> *어떻게 보여야 하는지*. 하네스 프레임워크 Layer 1 (선택이지만 **품질의 핵심**). 이 파일이 없으면 AI가 기본 스타일 슬롭을 뱉는다.

## 디자인 원칙

0. **충실도 우선(Track 1, ADR-014)** — 원본 **AgentCodeGUI 실제 디자인을 1:1로 따른다**: OKLCH 듀얼테마(라이트/다크)·**radius 12px**·소프트 섀도우·**모달 backdrop blur**·데스크톱 위 둥근 카드 셸. 아래 원칙·안티슬롭은 *그 안에서* 적용한다. **권위 스펙 = `docs/UI_FIDELITY.md`**(토큰·셸·격차). (이전의 "둥근모서리 절제/glass 금지"는 원본과 상충 → 원본 기준으로 대체.)
1. **IDE-grade 밀도** — 정보 밀도 높게. 여백 과다 금지. 개발자 도구는 화려함보다 한눈에 보이는 상태가 우선.
2. **레퍼런스 = AgentCodeGUI(원본)** — 소스/스크린샷 대조가 1순위. VS Code/JetBrains 관습은 보조.
3. **테마별 강조색** — 라이트=따뜻한 코랄(`--accent` oklch 0.61 0.16 42), 다크=니어-화이트(모노크롬). 기능색(추가/삭제/실행중/에러)은 상태 전달 전용.


> 🎯 **충실도 타깃(원본 실제 셸, `docs/UI_FIDELITY.md`)**: 위 3-pane는 단순화. 원본은 **데스크톱 위 둥근 카드** 안에 **타이틀바 + 4컬럼**(① 사이드바=채팅목록 w248 / ② 탐색기 w236·접힘 rail / ③ 대화+컴포저+파일탭 / ④ 에이전트 패널 w392) + 모달들 + ResizeHandles. 충실도 트랙 F1~F6에서 이 구조로 수렴.

- **좌측**: 파일 트리. AI가 건드린 파일은 인디케이터(점/색). 클릭 → 중앙 옆 코드뷰어/diff 탭.
- **중앙**: 대화. 도구호출은 *접이식 카드*(역할·도구명·입력·결과). 사용자 입력창 하단 고정.
- **우측**: 현재/큐 작업, 변경 파일, 서브에이전트 검사 카드. MVP는 단일 작업.
- **Git 패널**(B-tier): fork 스타일 3컬럼(히스토리·브랜치 / 커밋목록 / 커밋상세) — 별도 뷰/모달.

## 색상 팔레트 — OKLCH 듀얼테마 (원본 1:1, ADR-014)

> ⚠️ 기존 hex 토큰(`--bg-0` 등)은 **충실도 트랙 F1에서 OKLCH로 대체**된다. **권위 값 = `docs/UI_FIDELITY.md`**(라이트 `:root` + 다크 `:root[data-theme="dark"]` 전체 토큰). 테마 전환은 `document.documentElement[data-theme]` 한 줄.

| 토큰(요지) | 라이트 | 다크 | 용도 |
|---|---|---|---|
| `--desktop` | `oklch(0.90 0.006 75)` | `oklch(0.145 0 0)` | 카드 바깥 데스크톱 |
| `--bg` | `oklch(0.992 0.002 80)` | `oklch(0.196 0 0)` | 메인 카드/패널 |
| `--surface`(+1~+3) | `0.972 → 0.915` | `0.228 → 0.312` | elevation 단계 |
| `--inset` | `oklch(0.978 0.003 80)` | `oklch(0.168 0 0)` | 입력/코드 배경 |
| `--text`(4단계) | `0.27 → 0.69` | `흰색 0.92 → 0.34` | 텍스트 강조도 |
| `--accent` | `oklch(0.61 0.16 42)` 코랄 | `oklch(0.90 0 0)` 니어-화이트 | 강조 |
| `--green/--red/...` | 기능색(저채도) | 밝은 변형 | 추가/삭제/실행/에러 |

- `--radius: 12px`, `--shadow-win`(큰 소프트섀도우), `--font-sans: 'Wanted Sans Variable'`, `--font-mono: 'JetBrains Mono'`.
- 코드 하이라이트=highlight.js + (M2-LSP) LSP 시맨틱 토큰. 언어별 팔레트(예: C# Rider) 오버라이드.

## 타이포그래피
- UI: 시스템 산세리프(Segoe UI 등).
- 코드/diff: 등폭(JetBrains Mono / Cascadia Code / Consolas 폴백).
- 본문 13–14px, 밀도 높은 IDE 스케일.

## 컴포넌트 패턴
- **도구호출 카드**: 헤더(아이콘+도구명+상태색) / 접이식 본문(입력·결과). 기본 접힘, 실행중·에러는 펼침.
- **diff 뷰**: 라인 추가=`--ok` 좌측 바, 삭제=`--del` + 취소선/배경, 변경줄 거터.
- **스트리밍**: 토큰 단위 append, 커서 깜빡임. 자동 스크롤(사용자 스크롤업 시 정지).
- **빈 상태**: "폴더를 여세요" 같은 명확한 1줄 + 액션 버튼.

## ⚠️ AI 슬롭 안티패턴 (원본에 *없는* 임의 슬롭만 금지)
> 충실도 우선(ADR-014): **원본이 쓰는 건 슬롭이 아니다.** 원본 `C:/Dev/AgentCodeGUI/src/renderer/src/styles.css` 대조가 기준.
- ✅ **허용(원본 사용)**: radius **12px** · 모달 **backdrop blur**(절제) · 소프트 워암 섀도우(`--shadow-win`) · OKLCH 듀얼테마.
- ❌ 보라색/무지개 **그라데이션 텍스트**.
- ❌ **네온 글로우**(소프트 섀도우 ≠ 네온).
- ❌ 이모지를 *기능* 아이콘으로 — 벡터 아이콘(`icons.tsx`류) 사용.
- ❌ 과한 애니메이션(≤150ms ease, 기능적인 것만).
- ❌ 중앙 정렬 히어로/랜딩페이지풍 레이아웃(이건 IDE다).
- ❌ **원본에 없는** 임의 장식 — 의심되면 원본 컴포넌트/CSS 대조.
