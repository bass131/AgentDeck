# Phase 03: P0 매트릭스 8행 초안 — 계약 §2 행 스키마 1회전 실증

## 🎯 목표

P01의 8행 후보를 계약 §2 행 스키마 그대로 매트릭스 초안으로 작성한다. 목적은 완성도가 아니라 **스키마 실증** — 행 스키마·엔진별 상태/증거 분리·판정 초안 흐름이 실제로 굴러가는지 8행 비용으로 확인하고, 걸린 마찰을 기록하는 것이 진짜 산출물이다.

## ⏪ 사전 조건

- P01 완료. (P02와 의존성 없음 — **병렬 가능**.)

## 📝 작업 내용

1. 계약 §2 열 구성을 그대로 사용: 기능 / 상류 출처 / 표면 / 분류 / 상태-CC·상태-CX / 판정 / 증거-CC·증거-CX / 사이클.
2. 상류 출처는 §1 스냅샷 요건(버전 태그 + 조사 시점) 충족 표기.
3. 상태-CC/CX는 엔진별 독립 기입 — 전용 기능의 반대편 열은 "해당 없음". **✅는 같은 엔진의 증거 열이 채워진 경우만** (철칙 ①·④).
4. 판정 열은 전 행 "초안:" 접두 + 판정 근거 한 줄 + 재검토 조건(만료 트리거). 확정은 P04에서 영호가 한다.
5. 작성 중 걸린 스키마 마찰(열 의미가 애매한 지점·증거 형식 불명 등)을 별도 절 "스키마 마찰 기록"에 남긴다.
6. 산출물: `00.Documents/reports/LP1-03-P0-매트릭스-초안.md`

## 📚 학습 포인트

- 엔진별 증거 분리(철칙 ④)가 왜 필요한가 — 단일 엔진 증거로 행 전체를 플립하면 매트릭스 절반이 허수가 되는 문제 (Codex 검토 구멍 ②).
- 판정과 상태의 분리 — "존재하는가"는 기계, "위화감 없는가"는 사람 (work-judge 버킷 (a)/(b)와 동형).

## ⚠️ 함정

- 상태 "없음"에는 증거 의무가 없다 — 증거 의무는 ✅ 플립에만 (철칙 ①). "없음"의 근거 좌표는 권장이되 공란 허용.
- Codex 열 실측 시: AgentDeck의 Codex 어댑터는 stub — 상태-CX 대부분 "없음"이 정상. **상류 Codex 자체의 기능 존재 여부(상류 출처 열)와 AgentDeck 구현 상태(상태-CX 열)를 혼동하지 말 것.**
- `FEATURE_MAP.md` 본편을 편집하지 말 것 — 초안은 별도 문서.
