# UI_GUIDE — AgentDeck

> *어떻게 보여야 하는지*. 하네스 프레임워크 Layer 1 (선택이지만 **품질의 핵심**). 이 파일이 없으면 AI가 기본 스타일 슬롭을 뱉는다.

## 디자인 원칙

1. **IDE-grade 밀도** — 정보 밀도 높게. 여백 과다 금지. 개발자 도구는 화려함보다 한눈에 보이는 상태가 우선.
2. **레퍼런스 = AgentCodeGUI / VS Code / JetBrains 계열** — 익숙한 IDE 관습을 따른다.
3. **무채색 베이스 + 1개의 강조색** — 색은 *상태 전달*에만(추가/삭제/실행중/에러).

## 레이아웃 — 3-pane (벤치마킹)

```
┌──────────────────────────────────────────────────────────────────────┐
│  타이틀바: [워크스페이스명]   [백엔드: Claude Code ▼]   [토큰 게이지]  ─ □ ✕ │
├────────────┬───────────────────────────────────┬───────────────────────┤
│  좌측       │            중앙                     │        우측            │
│  파일탐색기  │         대화 패널                   │     에이전트 상태       │
│  (+AI 인디) │  - 스트리밍 메시지                   │  - 진행중 작업/큐       │
│            │  - 도구호출 카드                     │  - 변경된 파일 목록     │
│  [diff/뷰어 │  - 입력창(첨부·슬래시)              │  - 서브에이전트 카드    │
│   탭 전환]  │                                     │                       │
├────────────┴───────────────────────────────────┴───────────────────────┤
│  하단 바: 브랜치  ·  변경 N  ·  엔진 상태  ·  알림                         │
└──────────────────────────────────────────────────────────────────────┘
```

- **좌측**: 파일 트리. AI가 건드린 파일은 인디케이터(점/색). 클릭 → 중앙 옆 코드뷰어/diff 탭.
- **중앙**: 대화. 도구호출은 *접이식 카드*(역할·도구명·입력·결과). 사용자 입력창 하단 고정.
- **우측**: 현재/큐 작업, 변경 파일, 서브에이전트 검사 카드. MVP는 단일 작업.
- **Git 패널**(B-tier): fork 스타일 3컬럼(히스토리·브랜치 / 커밋목록 / 커밋상세) — 별도 뷰/모달.

## 색상 팔레트 (다크 우선)

| 토큰 | 값(권장) | 용도 |
|---|---|---|
| `--bg-0` | `#0e0f12` | 앱 배경 |
| `--bg-1` | `#16181d` | 패널 |
| `--bg-2` | `#1e2127` | 카드/입력 |
| `--border` | `#2a2e37` | 경계선 |
| `--text-0` | `#e6e8ec` | 본문 |
| `--text-1` | `#9aa0aa` | 보조 |
| `--accent` | `#4c8dff` | 강조(단 하나) |
| `--ok` | `#3fb950` | 추가/성공 |
| `--del` | `#f85149` | 삭제/에러 |
| `--run` | `#d29922` | 실행중 |

- 라이트 테마는 동일 토큰의 라이트 매핑(마일스톤 03). 토큰은 CSS 변수 한 곳에서.
- 언어별 JetBrains 컬러 스킴 자동 적용(코드뷰어, B-tier).

## 타이포그래피
- UI: 시스템 산세리프(Segoe UI 등).
- 코드/diff: 등폭(JetBrains Mono / Cascadia Code / Consolas 폴백).
- 본문 13–14px, 밀도 높은 IDE 스케일.

## 컴포넌트 패턴
- **도구호출 카드**: 헤더(아이콘+도구명+상태색) / 접이식 본문(입력·결과). 기본 접힘, 실행중·에러는 펼침.
- **diff 뷰**: 라인 추가=`--ok` 좌측 바, 삭제=`--del` + 취소선/배경, 변경줄 거터.
- **스트리밍**: 토큰 단위 append, 커서 깜빡임. 자동 스크롤(사용자 스크롤업 시 정지).
- **빈 상태**: "폴더를 여세요" 같은 명확한 1줄 + 액션 버튼.

## ⚠️ AI 슬롭 안티패턴 (금지)
- ❌ Glass morphism(반투명 블러) 남용.
- ❌ 보라색/무지개 **그라데이션 텍스트**.
- ❌ 네온 글로우·과한 box-shadow.
- ❌ 의미 없는 둥근 모서리 과다(IDE는 4–6px 절제).
- ❌ 이모지를 기능 아이콘으로(상태는 색/벡터 아이콘으로).
- ❌ 과한 애니메이션(150ms 이내 ease, 기능적인 것만).
- ❌ 중앙 정렬 히어로/랜딩페이지풍 레이아웃(이건 IDE다).
