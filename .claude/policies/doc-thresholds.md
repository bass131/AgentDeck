# Doc Thresholds — 문서 세분화 정책 (220줄·350줄)

> **헌법 참조**: 본 정책은 헌법(`../../CLAUDE.md`) "문서 운영 / 문서 세분화" 섹션에서 링크됩니다.
> 충돌 시 헌법이 이깁니다.

본 문서는 `.md` 파일이 비대해질 때의 *세분화 절차*를 정의합니다. 임계 자체는 유지하되, 등급 체계([`grade-and-risk.md`](grade-and-risk.md))와 정합.

> **본 정책이 외부화된 이유**: 정책 본문이 *헌법에 박혀있을 때* 헌법이 220줄 임계의 *주체이자 대상*이 됨 → 매 세션 "응축 제안" 자기참조 루프 위험. 외부화로 헌법은 본 정책의 *고객*이 되어 자기참조 해소.

---

## 1. 220줄 임계 (기본 룰)

`.md` 파일이 **220줄을 넘으면** 문서 종류에 따라 응답:

| 문서 종류 | 응답 | 예시 |
|---|---|---|
| 누적 섹션 있음 | 그 섹션을 별도 파일로 외부화. 원본은 참조 링크만. | `00.Documents/ADR.md` → 카테고리 외부화 |
| 응축 가능 | 재작성. 옛 디테일은 git/CHANGELOG로 위임. | work-pin (`current-pin.txt`) 자체 (30~40줄 목표 재작성) |
| 단위 작업 문서 | **자르지 않음**. 220줄 넘었다 = *작업 단위 등급이 너무 컸다*는 신호 → 등급 재산정 + `/work-plan`으로 더 잘게 분해. | Phase 파일, `-DONE.md` |
| 슬래시 커맨드 | **응축 가능형 동일 룰**. 단계별 외부화(별도 커맨드 파일로 위임)는 재사용성 있을 때만. | `.claude/commands/session/end.md` |

### 등급 체계 정합

단위 작업 문서 비대 시 *등급 재산정*이 핵심 — 220줄 넘었다는 건:

- 단순/보통 등급으로 시작했는데 실제론 복잡/대규모였다 → 등급 자동 상향 + Phase 분해
- 복잡으로 시작했는데 더 큰 작업이었다 → 대규모로 상향 + coordinator 위임

등급 정의·자동 상향 트리거 → [`grade-and-risk.md`](grade-and-risk.md).

## 2. 분리된 파일도 220줄 넘으면 (재귀 세분화)

### Level 1 (단일 파일) → Level 2 (주제별 카테고리)

```
{원본}/
├── INDEX.md            ← 한 줄 요약 + 링크
├── {topic-1}.md
└── {topic-2}.md
```

### Level 2 → Level 3 (비대해진 카테고리만 사건별)

**일률 적용 X**. 비대해진 카테고리만 Level 3로. 다른 카테고리는 그대로.

---

## 3. 폴더 분류 기준

| | 기준 | 예시 |
|---|---|---|
| ❌ | **단위 기준** (시간 분기, 마일스톤 묶음) | `2026-Q2.md`, `M1.md` |
| ✅ | **항목 기준** (주제 카테고리) | `phases.md`, `decisions.md` |

**이유**: 항목 기준이 *내용 단위*라 검색·참조가 직관적이고, 비대 시 사건별로 재귀 세분화가 자연스러움.

---

## 4. INDEX.md 규칙

분리된 파일들이 모이는 폴더에는 **항상 `INDEX.md`** 둠:
- 시간순(또는 의미순) 한 줄 요약 + 링크
- 새 항목 추가 시 INDEX.md도 함께 갱신

본 폴더 자체가 사례: [`INDEX.md`](INDEX.md).

## 5. 헌법(`CLAUDE.md`) 예외 — 350줄 임계

`CLAUDE.md`는 220줄이 아닌 **350줄 임계**로 예외 처리.

### 이유

- **자기참조 무한 제안 루프 차단** — 헌법이 220 임계의 주체이자 대상이라 매 세션 "응축 제안" 무한 발화 위험
- 절대 원칙 + 사용자 컨텍스트 + 새 절(등급/SubAgent)이 모인 *본질적으로 큰 정책 모음*이라 220줄 이하 강제는 핵심 룰 손상 위험
- 350줄은 헌법의 자연스러운 성장 여지 + 진짜 비대 시 발화점 둘 다 확보

### 적용 범위

이 예외는 **`CLAUDE.md`에만 적용**. 다른 사전형 문서(`00.Documents/ARCHITECTURE.md` 등)는 220줄 임계 그대로 — 필요 시 별도 ADR로 예외 박기.

### 단, 외부화 가능하면 외부화가 우선

본 정책 자체가 헌법에서 외부화됨. **350줄 안 넘어도** 정책·운영 가이드 성격이면 본 폴더(`policies/`)로 외부화 권장. 헌법은 절대 원칙·라우팅·등급/모델 분담만 남기는 게 본질.

## 6. 발동 체크리스트 (사용자 또는 AI가 비대 감지 시)

1. **종류 식별** — 누적형 / 응축 가능형 / 단위 작업형 / 슬래시 커맨드 중 무엇?
2. **단위 작업형**이면 → *등급 재산정* + `/work-plan`으로 분해
3. **외부화 또는 응축 또는 분해** — 표에 따라 결정
4. **링크 갱신** — 원본에서 외부 파일로 *참조 링크* 박기
5. **INDEX.md 갱신** — 분리된 파일이 들어간 폴더에
6. **헌법 참조 점검** — 헌법이 이 문서를 가리키고 있다면 그 링크도 갱신
7. **동기화 책임 점검** — 본 정책의 [7번 절] 의존성 목록 확인
