H1 Codex Harness Hardening

Claude Harness를 보존하면서 Codex 실행 계약, Hook, 권한, 역할별 모델 비용 계층과 검증기를 정합화했다.

무엇을 만들었나

메인 Full Access와 SubAgent 최소 권한 프로필, 네 lifecycle Hook, Sol·Terra·Luna 역할 매핑, static/live를 분리한 Harness doctor를 구축했다.

왜 필요한가

문서에 적힌 역할과 실제 런타임이 어긋나거나 Hook 실패 배너가 반복되는 상태에서는 안전성과 비용 절감 모두 검증할 수 없기 때문이다.

어떻게 만들었나

  • root는 :danger-full-access, SubAgent는 read-only/workspace/operations로 분리했다.
  • stale Hook digest는 실패 대신 조용히 no-op하고 doctor가 재신뢰 필요를 판정한다.
  • .codex/state write 승격으로 Windows sandbox가 깨지는 충돌을 제거했다.
  • native custom agent selector와 model telemetry 부재는 성공으로 위장하지 않고 degraded로 기록했다.
