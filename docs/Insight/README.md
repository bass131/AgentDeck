# Insight — 외부 벤치마킹·학습 인사이트

> **역할**: 외부 코드베이스(특히 벤치마킹 대상 [AgentCodeGUI](https://github.com/UnrealFactory/AgentCodeGUI))를
> 정독·비교하며 얻은 **설계 철학·코드 컨벤션 인사이트**를 박제하는 폴더. `docs/`의 다른 문서가
> *우리의 결정*(PRD·ARCHITECTURE·ADR·UI)을 다룬다면, 여기는 *남의 코드에서 배운 것 + 우리와의 격차*를 다룬다.
>
> **왜 분리?** ADR은 "우리가 무엇을 왜 결정했나"의 기록이고, FEATURE_MAP은 "기능 복제 추적표"다.
> 둘 다 "컨벤션/철학 수준의 정성적 학습"을 담기엔 결이 안 맞아서, 학습 산출물 전용 공간을 둔다.
> 여기 문서는 **시점 스냅샷**(특정 커밋 기준)이며, 결정이 아니라 *관찰*이다 — 액션으로 승격하려면 ADR/Phase로 따로 올린다.

## 작성 규칙
- 파일명: 주제 기준 `UPPERCASE_SNAKE.md` (docs/ 루트 컨벤션 일치). 시점은 파일명이 아니라 **문서 헤더 메타**(대상 커밋·작성일)에 박는다.
- 모든 비교 주장은 **`파일:라인` 근거**를 단다. "좋아 보인다"식 인상비평 금지 — 코드가 증거.
- 판정은 4분류로: **parity**(대등) / **learn**(우리가 배울 것) / **ahead**(우리가 앞선 것) / **weakness**(우리 약점).
