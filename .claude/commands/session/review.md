[`/session:start`](start.md)(구현 세션)와 **짝** 커맨드. 본 슬래시는 **리뷰용(pull) 세션**입니다. 루프가 구현을 빠르게 처리하는 동안 *미뤄둔 깊은 학습·점검*을 여기서 몰아 봅니다 ([`loop-driver.md`](../../policies/loop-driver.md) §6: 구현과 학습 분리).

> **왜 분리?** 구현 세션은 "흐름 안 끊김"이 목표라 깊은 학습을 끼우면 처리량이 깎입니다. 그렇다고 학습을 버리면 button-pusher가 됨(영호는 engineer로 남아야 함). 그래서 *별도 pull 세션*으로 — 영호가 *원할 때* 깊게 판다.

---

### 0. git 가벼운 점검

리뷰 세션은 보통 코드를 안 바꿉니다. `git status -sb`로 **현재 브랜치 + dirty 여부만 알림**(작업 풀게이트 아님). 리뷰 중 실제 수정이 생기면 그때 [`/session:start`](start.md) 작업 흐름으로 전환.

### 1. 깊게 볼 항목 선정

AgentDeck은 별도 pending-comprehension 원장을 두지 않습니다. 대신 *"깊게 안 본 항목"* 후보를 영호에게 제시:
- 최근 `.claude/CHANGELOG.md` [H]/[M] 변경
- 최근 commit / `-DONE.md`에서 "어떻게/왜"를 안 짚고 넘어간 것
- work-pin "현재 작업" 인접 개념

```
📚 깊게 볼까 싶은 항목:
  1. [최근] 엔진 추상화 AgentBackend — 어댑터 패턴이 왜 이렇게?
  2. [최근] IPC 계약 버전 협상 — 어떻게 동작하나
  ...
어느 걸 깊게 볼까요? (번호 / 직접 지정 / "오늘은 패스")
```

### 2. 깊은 설명 모드 (학부생 멘토링)

영호가 항목을 고르면:
- **file:line 실측 먼저** (추측 X) → "어떻게 / 왜 이렇게 구현했나" + 트레이드오프 + 안 고른 대안.
- 구현 세션의 *흐름 안 끊김*과 **반대** — 여기선 충분히 멈춰서 깊게. 확인 질문으로 이해 점검.

### 3. 톤

멘토링·학습 집중. 구현 재촉 X. "이해했어" 답엔 핵심 개념 확인 질문 (헌법 사용자 컨텍스트).
