# Phase 04: 판정 초안 → 영호 확정 — Human door 1회전

## 🎯 목표

P03 매트릭스의 판정 초안 8행을 영호에게 일괄 제출해 행별 확정을 받고, 확정 스탬프를 반영한다. 계약 §3 Human door ①(에이전트 초안 제안 허용 · 확정은 영호 단독)의 첫 실증이다.

## ⏪ 사전 조건

- P03 완료 (`LP1-03-P0-매트릭스-초안.md` 8행 + 판정 초안).

## 📝 작업 내용

1. 8행 판정 초안을 요약 표로 정리해 영호에게 제출 (행당: 기능 / 초안 판정 / 근거 / 재검토 조건).
2. 영호가 행별로 승인 / 수정 / 보류 결정 — **여기서 루프는 정지하고 사람 응답을 기다린다** (loop_track: human-gate).
3. 확정 결과를 `LP1-03-P0-매트릭스-초안.md` 판정 열에 반영: "초안:" 제거 → "확정: 영호 YYYY-MM-DD". 수정된 행은 수정 전후 값과 사유 1줄을 기록.
4. 판정 과정에서 드러난 마찰(초안 양식이 판단에 불충분했던 지점 등)을 "스키마 마찰 기록" 절에 추가.

## ✅ 완료 조건 (정량)

- [ ] 8행 전부 판정 열에 "확정: 영호 YYYY-MM-DD" 스탬프 — "초안:" 라벨 잔존 0.
- [ ] 수정 행 전부에 수정 전후 값 + 사유 1줄.
- [ ] 판정 마찰 기록 갱신 (0건이면 "0건" 명기).

## 📚 학습 포인트

- Outer Loop Verdict — 판정 *권한*은 사람에게 남기면서 기입 *비용*만 에이전트로 내리는 설계. 병목 완화와 권한 이전은 다르다.

## ⚠️ 함정

- 영호 부재 시 이 Phase에서 **정지** — 대리 확정·"아마 승인일 것" 진행 금지 (철칙 ②: 판정 열은 사람 전용).
- 확정 반영은 판정 열만 — 상태·증거 열을 이 Phase에서 건드리지 않는다.
